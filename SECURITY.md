# Security Policy

## Supported Versions

Only the latest release version on the `main` branch is actively supported with security updates. We recommend always running the latest version.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of Ravanshenasi seriously. If you find a security vulnerability, please do **not** report it publicly via GitHub issues. Instead, report it privately to our security team.

### How to report:

1. Send an email to **security@ravanshenasi.com**.
2. Include a detailed description of the vulnerability, steps to reproduce, and any proof of concept or potential impact.
3. If possible, encrypt your email or share details in a secure format.

### Our Commitment:

- We will acknowledge receipt of your vulnerability report within 48 hours.
- We will keep you updated on our progress as we investigate and develop a fix.
- We will coordinate public disclosure with you to ensure a responsible disclosure timeline.
- Credits will be provided to reporters who follow this policy in security advisory notes.
